# Binmon
Short for "Binary Monitor", it's a small project where I plan on using C and potentially ebpf or other related technologies to gather telemetry about different running processes on a Linux system. With these processes, I can determine if they need to be "optimized" either by removal or replacement. But the "Binary" part is a part of my main vision which is to search for binaries or just random things installed on the system and use data such as "how long ago was this used", or "when was this downloaded", to determine if it can be removed or not. I have this feeling where If im using some sort of Linux system for a long time, that i've downloaded random stuff in the past without removing them. So I want a programmatic approach to detect these long-gone toolings and just remove them. But this is also a sort of IDS system, in a way, just for endpoints as well.
